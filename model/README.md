# symbol-model

线谱符号 CNN 识别后端。本文档定义**分类标签体系**与**网络架构**，作为后续实现的规范。

---

## 1. 任务定义

| 项目 | 说明 |
|------|------|
| 任务类型 | 孤立符号图像分类（单张裁剪图 → 一个类别） |
| 符号体系 | 线谱（五线谱）符号，见 `enum.ts` → `MUSIC_SYMBOL_S` |
| 输入前提 | 已完成符号检测与裁切；本模型不负责整页识谱 |
| 类别总数 | **34**（`MUSIC_SYMBOL_S_COUNT`） |

---

## 2. 分类标签设计（`MUSIC_SYMBOL_S`）

### 2.1 音符（17 类）

| 时值 | 类别 | 说明 |
|------|------|------|
| 全音符 | `WHOLE_NOTE` | 无符干，不区分朝向 |
| 二分～二百五十六分 | `*_NOTE_STEM_UP` / `*_NOTE_STEM_DOWN` | 符干朝上、朝下各一类 |

**符干方向拆类的原因：** 项目目标包含识别符干朝向，而非仅识别时值。二分音符及更短音符均区分 `STEM_UP` / `STEM_DOWN`。

时值覆盖：全、二、四、八、十六、三十二、六十四、一百二十八、二百五十六分。

### 2.2 休止符（9 类）

与音符时值一一对应，共 9 类：`WHOLE_REST` … `TWO_HUNDRED_FIFTY_SIXTH_REST`。

休止符无符干，不区分朝向。

### 2.3 谱号（3 类）

`TREBLE`（高音谱号）、`BASS`（低音谱号）、`ALTO`（中音谱号）。

### 2.4 变音符号（5 类）

`NATURAL`、`SHARP`、`FLAT`、`DOUBLE_FLAT`、`DOUBLE_SHARP`。

### 2.5 类别索引

枚举从 `0` 连续编号至 `33`，与 Softmax 输出维度一一对应。完整列表见 `enum.ts`。

## 3. CNN 架构

### 3.1 总览

```
输入图像 96×96 灰度
    ↓
[Conv 32, 3×3, pad=1] → ReLU → [MaxPool 2×2, stride=2]   → 48×48
    ↓
[Conv 64, 3×3, pad=1] → ReLU → [MaxPool 2×2, stride=2]   → 24×24
    ↓
[Conv 128, 3×3, pad=1] → ReLU → [MaxPool 2×2, stride=2]  → 12×12
    ↓
Global Average Pooling                                    → 128 维
    ↓
Dense 128 → ReLU → Dropout(0.5)
    ↓
Dense 34 → Softmax
```

### 3.2 输入规范

| 参数 | 值 | 说明 |
|------|-----|------|
| 尺寸 | **96 × 96** | 固定；所有符号统一 resize 到此尺寸 |
| 通道 | **1**（灰度） | 线谱符号为黑白线条，灰度足够 |
| 数值范围 | `[0, 1]` | 像素归一化 |
| 裁切 | 单符号 + 边距 | 裁切框四周留 10%～20% 空白，避免符干、符尾被截断 |

**为何选 96×96：** 比 64×64 更能保留谱号、加线音符、密集符尾等细节；参数量仍可接受，适合 34 类符号分类。

### 3.3 卷积层

| 层 | 滤波器数 | 核大小 | padding | stride | 激活 |
|----|----------|--------|---------|--------|------|
| Conv1 | 32 | 3×3 | 1 | 1 | ReLU |
| Conv2 | 64 | 3×3 | 1 | 1 | ReLU |
| Conv3 | 128 | 3×3 | 1 | 1 | ReLU |

**设计说明：**

- **3×3 核：** 适合捕捉线条、弧、符头、符干等局部笔画。
- **padding=1：** 卷积后保持空间尺寸，避免边缘笔画（符干顶端、谱号弧线）过早丢失。
- **stride=1（卷积）：** 下采样仅由池化完成，不在卷积层做降采样。
- **通道 32→64→128：** 逐层提取从边缘到形状组合的抽象特征。

### 3.4 池化层

| 层 | 类型 | 窗口 | stride | 每次后尺寸 |
|----|------|------|--------|------------|
| Pool1 | MaxPool | 2×2 | 2 | 96 → 48 |
| Pool2 | MaxPool | 2×2 | 2 | 48 → 24 |
| Pool3 | MaxPool | 2×2 | 2 | 24 → 12 |

**设计说明：**

- **MaxPool** 比 AvgPool 更适合线状笔画：保留最强边缘响应。
- 三次池化将 96 降至 12，在感受野与计算量之间折中。

### 3.5 全连接与输出

| 层 | 输入维 | 输出维 | 激活 / 正则 |
|----|--------|--------|-------------|
| GAP | 12×12×128 | 128 | — |
| Dense | 128 | 128 | ReLU |
| Dropout | — | — | p = 0.5 |
| Dense（输出） | 128 | **34** | Softmax |

**设计说明：**

- **Global Average Pooling（GAP）：** 将 12×12×128 压为 128 维，减少参数量，降低过拟合；对裁切框略大略小更宽容。
- **全连接 1 层（128 维）：** 对 34 类符号分类足够；不必堆叠多层全连接。
- **Dropout 0.5：** 34 类、样本有限时抑制过拟合。

### 3.6 参数量（估算）

| 模块 | 参数量（约） |
|------|-------------|
| Conv1 (1→32) | 320 |
| Conv2 (32→64) | 18,496 |
| Conv3 (64→128) | 73,856 |
| Dense (128→128) | 16,512 |
| Dense (128→34) | 4,386 |
| **合计** | **~11 万** |

可训练规模适中，适合本地训练与推理。

## 4. 数据与训练规范（待实现时遵循）

### 4.1 样本量建议

| 阶段 | 每类最少样本 | 说明 |
|------|-------------|------|
| 原型验证（13 类子集） | 200+ | 先验证流程 |
| 完整 34 类 | 500+ | 少样本类需加强增强或过采样 |

### 4.2 数据增强

| 增强 | 适用范围 | 说明 |
|------|----------|------|
| 小角度旋转 | 全部 | ±5°～10°，模拟扫描倾斜 |
| 平移 | 全部 | ±5% 画布，模拟裁切偏差 |
| 缩放 | 全部 | 0.9～1.1，模拟尺寸差异 |
| 笔画粗细变化 | 全部 | 模拟不同字体 / 扫描质量 |
| 垂直翻转 | **慎用** | 仅当标签含 `STEM_UP`/`STEM_DOWN` 时需成对替换标签，不可无脑翻转 |

### 4.3 损失函数与优化

- **损失：** 交叉熵（Categorical Cross-Entropy）
- **优化器：** Adam（lr 建议 1e-3 起，按验证集调整）
- **评估指标：** 准确率、混淆矩阵（关注时值相近、符干上下成对的类别）

### 4.4 符干方向相关样本

`STEM_UP` / `STEM_DOWN` 成对类别需在训练集中**均衡采集**：

- 同一时值、朝上/朝下样本数量尽量相当
- 不同谱表位置（线上、间上）均应覆盖
- 避免模型只学到「符干在图的哪一侧」而忽视符号本体

## 5. 图像预处理管线

### 5.1 流程

```
file 或 arr（arr 优先）
    ↓
解码为灰度矩阵 [0,1]（0=笔画，1=背景）
    ↓
裁掉四周全白边（灰度 ≥ 0.95 视为白）
    ↓
非正方形 → 向右或向下补白成正方形
    ↓
等比缩放到 96×96
    ↓
供 CNN 使用
```

### 5.2 输入来源

| 来源 | 处理方式 |
|------|----------|
| **file** | `sharp` 解码 png/jpg/svg/webp → 灰度；SVG 先栅格化（density 192） |
| **arr** | 直接解析二维数组，**不走 SVG 绕路**；支持 0/1 或 0~255 |

**arr 约定：** `0`=背景（白），`1`=笔画（黑），服务端转为 CNN 格式（0=笔画，1=背景）。

**arr 缩放：** 最近邻（`nearest`），保持二值边缘清晰。  
**file 缩放：** Lanczos3，适合带抗锯齿的栅格图。

### 5.3 裁边与补方

- 从外向内删除**整行/整列均为白色**的边（阈值 ≥ 0.95）
- 裁后若宽 > 高：向下补白行；若高 > 宽：向右补白列
- 补方后等比缩放至 96×96（不变形）

## 6. HTTP 接口

### 6.1 连通性

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/symbol/test` | 返回 `{ "text": "接口访问成功" }` |

### 6.2 符号检测 / 训练

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/msd/detect` | 预处理 + 预留 CNN 推理 |
| POST | `/msd/train` | 预处理 + 预留 CNN 训练写样本 |

**请求格式：** `multipart/form-data`

| 字段 | 必填 | 说明 |
|------|------|------|
| `notation` | 是 | `s`=线谱（`MUSIC_SYMBOL_S`），`n`=简谱（`MUSIC_SYMBOL_N`） |
| `file` | 二选一 | png / jpg / svg / webp |
| `arr` | 二选一 | JSON 字符串，二维 0/1（或 0~255）数组 |
| `label` | 否 | 仅 `/msd/train`；非负整数类别索引 |

**优先级：** `arr` 与 `file` 同时存在时，**优先处理 arr**。

**当前响应：**

| 接口 | 响应 |
|------|------|
| `/msd/detect` | `{ ok, notation, label, labelName, probs }` |
| `/msd/train` | `{ ok, notation, label, labelName, expectedLabel, loss, probs, weightsFile }` |

可选字段 `lr`（训练学习率，默认 `0.001`）。

### 6.3 权重持久化

线谱与简谱使用**独立权重文件**，互不影响：

| 文件 | 谱式 | 类别数 |
|------|------|--------|
| `weights-s.json` | 线谱（`notation=s`） | 34 |
| `weights-n.json` | 简谱（`notation=n`） | 1（占位，待补全 `MUSIC_SYMBOL_N`） |

- 服务启动时自动加载；文件缺失或无效则随机初始化并写入
- 每次 `/msd/train` 反向传播后**立即更新**对应 JSON 文件
- 实现位置：`lib/cnn/`（`model.ts` 网络、`weightsStore.ts` 读写）
