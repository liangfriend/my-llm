# 权重算法

本文档描述 melody-model 生成流程中的权重计算规则。  
可调系数集中在 `lib/weightConfig.ts`，实现见 `lib/weight.ts`、`lib/sampleFilter.ts`、`lib/melodyLine.ts`。

---

## 1. 通用约定

### 1.1 多维度合成（预留）

当多个维度同时参与权重计算时，使用**相乘**合成：

```
总权重 = 维度₁ × 维度₂ × …
```

未传请求参数时，该维度不参与计算（等价于系数 `1`）。  
实现：`combineDimensionWeights()`。

### 1.2 中位数过滤

第 2、4、5 步在算出权重数组后，取**中位数**作为阈值，保留 `权重 >= 中位数` 的项。  
若一轮过滤后数量未减少（防止权重全相等死循环），则停止继续过滤。

实现：`median()`（`lib/sampleFilter.ts`）。

---

## 2. 第 2 步：样本过滤

### 2.1 minMidi / maxMidi（硬过滤，非权重）

| 条件 | 行为 |
|------|------|
| 请求未传 minMidi 且未传 maxMidi | 不做音高过滤 |
| 样本未声明 minMidi/maxMidi | 保留 |
| 请求显式传入边界 | 请求范围须**包裹**样本范围 |

包裹判定（样本 `sMin`/`sMax`，请求 `rMin`/`rMax`）：

- 若传了 `rMin`：要求 `rMin <= sMin`（样本缺省 `sMin=1`）
- 若传了 `rMax`：要求 `sMax <= rMax`（样本缺省 `sMax=128`）

不满足则**直接剔除**样本，不参与后续权重。

### 2.2 params（唯一权重维度）

请求传了 `params` 时，对每个样本计算标签相似度权重；再按中位数过滤。  
请求未传 `params` 时，跳过权重与中位数过滤，midi 过滤后的样本全部保留，权重记为 `1`。

#### 计算规则（沿用旧版 probabilityModel）

对请求 `params` 的**每个键**依次乘系数，初始值 `1`：

| 情况 | 乘数（配置键） | 默认值 |
|------|----------------|--------|
| 样本缺少该键 | `missingKeyFactor` | 0.7 |
| 字符串，相等 | `stringMatchFactor` | 1.6 |
| 字符串，不等 | `stringMismatchFactor` | 0.5 |
| 数字 | `1 / (1 + \|expected - actual\|)` | — |
| 其他类型，相等 | `genericMatchFactor` | 1.2 |
| 其他类型，不等 | `genericMismatchFactor` | 0.8 |

实现：`calcParamsSimilarityWeight()`。

#### 示例

请求：`{ style: "classical", mood: "calm" }`  
样本：`{ style: "classical", mood: "calm" }`  

权重 = `1.6 × 1.6 = 2.56`

请求：`{ style: "pop" }`  
样本：`{ style: "classical", mood: "calm" }`  

权重 = `0.5`（style 字符串不等）

## 3. 第 3 步：样本音符数组（已实现）

遍历第 2 步 `filteredSamples`，将每个样本的每句、每个音符平铺为 `sampleNoteEntries`。
每条记录包含：

- 所属样本标签权重 `sampleWeight`（来自第 2 步）
- 当前音：midi、chronaxie、拼音
- 本句：旋律线、总时值、音符数、歌词数
- 句内上一音：midi、chronaxie、拼音（句首为 null）
- 上一句：旋律线、总时值、音符数、歌词数、句尾音 midi/chronaxie/拼音（样本首句为 null）

同步建立等长 `sampleNoteWeights`，**本步初始全为 0**；具体权重在第 5 步对照生成上下文时再赋值。

实现：`buildSampleNoteIndexStep3()`（`lib/sampleNoteIndex.ts`）。

---

## 4. 第 4 步：目标样本句（下一句）

遍历第 3 步 `sampleSentencePairs`，为每个候选**下一句**计算权重并加权随机选取。

### 4.1 硬过滤

| 条件 | 行为 |
|------|------|
| 请求传了 `totalChronaxie` | 剔除 `pair.next.totalChronaxie < 参数` 的项 |
| 未传 `totalChronaxie` | 不做时值下限过滤 |

### 4.2 权重（累加）

```
总权重 = sampleWeight
       + totalChronaxie 对照分
       + 音符数对照分（可选）
       + preSentence 句尾对照分（可选）
```

| 维度 | 对照对象 | 加分 |
|------|----------|------|
| 样本标签 | 第 2 步 `sampleWeight` | 作为基数累加 |
| 下一句 totalChronaxie | `pair.next` vs 参数 `totalChronaxie` | 相等 **+10**；差 **16 → +4**；差 **32 → +2**；其余 **+1** |
| 下一句音符数 | `pair.next.sentence.length` vs `targetNoteLength` | 相等 **+2** |
| preSentence 句尾 | 参数 `preSentence` 末音 midi vs 样本**当前句**句尾 midi | 相同 **+5** |

未传 `totalChronaxie` 时，时值对照项固定 **+1**。

实现：`calcStep4PairWeight()`、`calcStep4TotalChronaxieWeight()`（`lib/weight.ts`）。

### 4.3 过滤与选取

1. 收集 `{ weight, next }` 候选
2. 中位数过滤（去掉 weight < 中位数）
3. 若候选数 > **10**，重复中位数过滤，直到 ≤ 10 或数量不再减少
4. 按 weight 加权随机选取，写入 `state.targetSampleSentence`

实现：`pickTargetSampleSentenceStep4()`（`lib/targetSampleSentence.ts`）。

---

## 5. 第 5 步：音符 midi / 时值（已实现）

循环直到 `generatedNoteCount >= targetNoteLength`。

### 5.1 硬过滤（样本音符数组）

| 规则 | 条件 |
|------|------|
| 休止符 | `canBeRest` 为 false 时剔除 `midi=0` |
| 音域 | 超出请求 `minMidi`/`maxMidi` 剔除 |
| 最大时值 | 超出 `getMaxChronaxieForCurrentNote` 剔除 |
| 时值间距 | 不符合 `(chronaxie - minChronaxie) % interval === 0` 剔除 |

若过滤后为空：**放宽间距规则**再滤；仍为空则 **降级**（`recommendMidiReplace` + 最小时值）。

### 5.2 权重对照（相乘）

| 维度 | 规则 |
|------|------|
| `sampleWeight` | 样本标签权重 |
| 句旋律线 | 与 `targetMelodyLine` 相似度 |
| 句音符数 / 歌词数 | 精确匹配 `targetNoteLength` / `text.length` |
| 句内上一音 | midi / 时值 / 拼音精确匹配 |
| 上一句（有 preSentence） | 旋律线相似 + 时值/音符数/歌词数/句尾音精确匹配 |
| 目标时值 | `1 / (1 + \|样本时值 - 目标时值\|)`，越接近越高 |

目标时值 = `expectedAverageChronaxie + chronaxieDrift`（首音 drift 为 0）。

### 5.3 选取与歌词

1. 中位数过滤直到候选 ≤ **6**（或无法再减）
2. 加权随机选取 midi、chronaxie
3. 歌词：`mustAddLyrics` 必赋；否则仅当样本音有拼音（曾有歌词）且还有剩余文本时赋值

实现：`generateNotesStep5()`（`lib/noteGenerator.ts`）。

### 5.4 差值权重

每生成一音后更新 `chronaxieDrift = 预期累加时值 - 当前累加时值`；下一音目标时值 ≈ `平均时值 + chronaxieDrift`。

---

## 6. 第 6 步：总时值调整（已实现）

仅当传入 `totalChronaxie` 且生成已有音符时执行。

```
剩余 = targetTotalChronaxie - currentAccumulatedChronaxie
```

| 剩余 | 行为 |
|------|------|
| ≤ 0 | 不调整（< 0 理论上不应出现） |
| > 0 | 若 `最后一音时值 + 剩余 < 256`，则并入最后一音 |
| 并入后 | 若不符合 minChronaxieInterval 规则，取 **≤ 合并值** 的最大合法时值 |

实现：`adjustTotalChronaxieStep6()`（`lib/chronaxieAdjust.ts`）。  
上限常量：`MAX_LAST_NOTE_CHRONAXIE = 256`（`lib/constants.ts`）。

---

## 7. 配置入口

所有默认乘数见 `lib/weightConfig.ts`：

```typescript
WEIGHT_CONFIG.params              // 第 2 步
WEIGHT_CONFIG.exactMatch          // 第 4、5 步精确匹配
WEIGHT_CONFIG.melodyLine          // 第 4 步旋律线相似度
WEIGHT_CONFIG.targetMelodyLine    // 第 4 步候选上限（maxCandidates: 10）
WEIGHT_CONFIG.noteGeneration      // 第 5 步候选上限（maxCandidates: 6）
```

修改后重新编译 / 重启 melody-model 即可生效。
